Navigation

Navigation patterns guide citizens between screens — back links for step-by-step flows, bottom navigation for top-level switching.

Component WCAG 2.1 AA

Back link

The primary navigation pattern within the application flow. Appears above the page <h1>. Text always names the destination screen — never "Back".

chevron_left Home chevron_left Personal details chevron_left Before you begin
chevron_left Personal details
check_circle
Do name the destination: "Personal details", "Home".
chevron_left Back
cancel
Don't use generic "Back" — screen reader users navigate by link text and "Back" is ambiguous.

Progress indicator

For multi-step forms, show the citizen's position in the flow. Display as a step count ("Step 2 of 5") above the page heading, not as a visual progress bar alone.

Employment

Section list (before you begin)

Used on the Before You Begin screen to preview all sections of the form. Sections use an inner card surface with a completed/pending state indicator.

radio_button_unchecked Employment Not started
radio_button_unchecked Income Not started

Rules

  • Back links navigate to a specific named screen — they do not call history.back(). Implement navigation state explicitly.
  • Back links are <a> elements (navigation to a page), not <button> elements (actions).
  • Section status must be communicated with text ("Complete", "Not started") — not colour alone.
  • Never put form steps or contextual actions in a bottom navigation bar — reserve the bottom nav for top-level destinations.
  • check_circle Wrap navigation in <nav aria-label="…">. Use distinct labels for multiple nav elements on the same page (e.g., "Breadcrumb", "Main navigation", "Step navigation"). 1.3.6 / 2.4.1 A
  • check_circle Back links must be <a href> elements with a meaningful label that names the destination. Screen reader users navigate by link text — "Back" alone is not descriptive. 2.4.4 AA
  • check_circle The leading chevron icon in the back link is decorative — apply aria-hidden="true". 1.1.1 A
  • check_circle Section completion status must include a text label ("Complete", "Not started"), not only a colour-coded icon. Icon and text together meet 1.4.1 (Use of Colour). 1.4.1 A
  • check_circle Step count ("Step 2 of 5") should be announced to screen readers. Place it in a <p> element with aria-label="Step 2 of 5" or make it part of the page <title>. 2.4.2 A
  • check_circle On single-page navigation (screen transitions without a full page reload), update the page <title> and move focus to the new <h1> or the top of the new screen content. 2.4.3 A

Back link

HTML
<nav aria-label="Step navigation">
  <a href="/personal-details" class="back-link">
    <span class="material-icons" aria-hidden="true">chevron_left</span>
    Personal details
  </a>
</nav>

HTML
<p class="step-count" aria-label="Step 2 of 5">
  Step 2 of 5
</p>
