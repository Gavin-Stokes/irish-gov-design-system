Form field

Form fields collect information from citizens. Every field must have a visible label, hint text where needed, and clear error messages.

Component WCAG 2.1 AA

States

Default

As it appears on your passport or driver's licence

Filled (pre-populated from MyGovID)

Error

error Enter a valid date of birth

Radio group

What is your current employment status?

Guidelines

  • Label every field visibly — never use placeholder text as a substitute for a label.
  • Position labels above fields, not beside them — this works better at mobile widths and is clearer for screen reader users.
  • Provide hint text for fields where the expected format may not be obvious (e.g. date format, PPSN format).
  • Pre-fill fields from MyGovID where available and clearly indicate the source.
  • Validate on blur (when the user leaves the field), not on every keystroke.
  • Group related fields with <fieldset> and <legend>.

Accessibility checklist

  • check_circle Every input must have a programmatically associated <label> via matching for / id attributes. Visual proximity alone is not sufficient. 1.3.1 A
  • check_circle Hint text and error messages must be linked to the input via aria-describedby. Multiple IDs can be space-separated: aria-describedby="hint error". 1.3.1 A
  • check_circle Error messages must include aria-invalid="true" on the input. Use role="alert" on the error message to announce it immediately. 4.1.3 AA
  • check_circle After a failed form submission, move focus to the error summary at the top, or to the first erroneous field. Users must not have to hunt for errors. 3.3.1 A
  • check_circle Use autocomplete attributes on personal data fields: given-name, family-name, email, tel, bday. This aids users with cognitive disabilities and motor impairments. 1.3.5 AA
  • check_circle Radio groups and checkbox groups must be wrapped in <fieldset> with a <legend>. The legend is the group question; individual options are labels. 1.3.1 A
  • check_circle Radio and checkbox tap targets must be full-width rows with 44px minimum height — not just the control itself. 2.5.5
  • check_circle Do not use disabled on form fields unless the field is truly unavailable for interaction. Use readonly when content should be visible but not editable. 4.1.2 A
  • check_circle Input borders must achieve 3:1 contrast against adjacent background. The default #888888 border on a white background achieves approximately 3.5:1 — acceptable but borderline. In error state, use #d4351c. 1.4.11 AA

Error summary pattern

When a form is submitted with errors, show a summary at the top of the form and move focus to it.

HTML
<div class="error-summary" role="alert" tabindex="-1">
  <h2>There is a problem</h2>
  <ul>
    <li><a href="#date-of-birth">Enter a valid date of birth</a></li>
    <li><a href="#ppsn">Enter your PPSN</a></li>
  </ul>
</div>

<script>
  // After failed submission
  const summary = document.querySelector('.error-summary');
  summary.focus();
</script>

Text input

HTML
<div class="form-group">
  <label class="form-label" for="first-name">First name</label>
  <span class="form-hint" id="first-name-hint">
    As it appears on your passport or driver's licence
  </span>
  <input
    class="form-input"
    type="text"
    id="first-name"
    name="firstName"
    autocomplete="given-name"
    aria-required="true"
    aria-describedby="first-name-hint"
  >
</div>

Error state

HTML
<div class="form-group">
  <label class="form-label" for="dob">Date of birth</label>
  <span class="form-error" id="dob-error" role="alert">
    <span class="material-icons" aria-hidden="true">error</span>
    Enter a valid date of birth
  </span>
  <input
    class="form-input form-input--error"
    type="text"
    id="dob"
    aria-invalid="true"
    aria-describedby="dob-error"
  >
</div>

Radio group

HTML
<fieldset>
  <legend class="form-label">
    What is your current employment status?
  </legend>
  <div class="radio-group">
    <label class="radio-label">
      <input type="radio" name="employment" value="employed">
      Employed
    </label>
    <label class="radio-label">
      <input type="radio" name="employment" value="self-employed">
      Self-employed
    </label>
    <label class="radio-label">
      <input type="radio" name="employment" value="unemployed">
      Unemployed
    </label>
  </div>
</fieldset>
