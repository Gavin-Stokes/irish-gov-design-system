Card

Cards group related information and actions into a visually distinct surface. Each card communicates one primary idea.

When to use

  • Use cards to surface distinct services, applications, or information blocks on the home screen.
  • Use inner cards to group checklist items or step progress within a larger card or section.
  • Use status cards when the application has a trackable state (submitted, in review, approved).

When not to use

  • Do not use cards purely for visual decoration — every card should contain actionable or informational content.
  • Do not stack more than three or four cards vertically without a break — consider a list or summary view instead.
  • Do not add borders to cards — elevation comes from the shadow only.
info
Cards must communicate their content to screen readers through proper heading structure and semantic HTML — the visual styling alone is not perceivable by assistive technology.

Accessibility checklist

  • check_circle Card headings must use the correct heading level for their position in the page hierarchy — typically <h2> for top-level cards, <h3> for nested cards. Never skip heading levels. 1.3.1 A
  • check_circle If the entire card is interactive (clickable), wrap it in a single <a> or <button>. Do not attach click handlers to <div> elements. 4.1.2 A
  • check_circle Decorative illustrations must have alt="" and aria-hidden="true". They add no information that isn't already conveyed by the card text. 1.1.1 A
  • check_circle Status pills that update dynamically (e.g. "In review" → "Approved") should be inside an aria-live="polite" region so screen readers announce the change. 4.1.3 AA
  • check_circle The background colour #EAF4F5 paired with near-black body text #0B0C0C achieves 16:1 contrast — well above the 4.5:1 AA minimum. 1.4.3 AA
  • check_circle Any interactive element inside a card (button, link) must individually receive keyboard focus — focus must not jump from the card container to outside it. 2.1.1 A

HTML
<div class="service-card">
  <!-- Decorative illustration -->
  <img src="../../assets/illustration-house.png"
       alt=""
       aria-hidden="true"
       class="service-card__illus">

  <p class="service-card__eyebrow">Current application</p>
  <h2 class="service-card__title">Social housing application</h2>
  <p class="service-card__body">
    Continue your application or check the status of a previous submission.
  </p>

  <button class="btn btn--primary" type="button">
    Continue
    <span class="material-icons" aria-hidden="true">arrow_forward</span>
  </button>
</div>

CSS
.service-card {
  background: var(--color-bg-card);      /* #EAF4F5 */
  border-radius: var(--radius-card-lg);  /* 10px */
  box-shadow: var(--shadow-card);
  padding: var(--card-padding);          /* 20px */
  position: relative;
  overflow: hidden;
}

.service-card__title {
  font-size: var(--text-h3);
  font-weight: var(--fw-bold);
  color: var(--color-text-heading);
  margin-bottom: var(--space-2);
}

.service-card__illus {
  position: absolute;
  top: 12px;
  right: 12px;
  width: 48px;
  height: 48px;
}

/* Inner card (nested) */
.inner-card {
  background: var(--color-bg-card-inner); /* #F0F0F0 */
  border-radius: var(--radius-card);      /* 8px */
  padding: 14px 16px;
  /* No shadow on inner cards */
}
