Getting started

Set up the design system in your project — from CSS tokens to React components.

Figma source file

The design source is Social Housing — Part 2 — DEPR.fig. The Figma file contains:

  • Assets page — all shared components, icons, and illustrations
  • Prototypes-Final — 15 mobile screens (390px wide) representing the full application flow
  • Personas, As-Is Journey, Service Blueprint — internal research artefacts, not UI components

Design tokens in Figma

Colour styles, text styles, and effect styles in Figma mirror the CSS custom properties in colors_and_type.css. When creating new screens:

  • Always apply colour styles from the shared library — do not use local fills with raw hex values.
  • Use text styles for all type — do not override size, weight, or colour individually.
  • Components are published from the Assets page — use instances, do not detach unless absolutely necessary.

Canvas and frame setup

  • All app screens use a 390×844px frame (iPhone 14 Pro size).
  • Horizontal margin: 16px each side.
  • Status bar height: 24px.
  • Use Auto Layout on all components for consistent spacing.
palette
All colour values, type styles, and spacing are documented in the Foundations section of this site with the exact Figma style names and CSS token equivalents.

Handoff checklist

  • All layers named clearly (not "Rectangle 47")
  • Colours and text applied via shared styles only
  • Spacing uses 8px grid — check with Layout Grid overlay
  • Interactive states (focus, hover, error) included for all interactive elements
  • Accessibility annotations added: heading levels, ARIA roles, alt text notes for images
  • Contrast checked using the Figma Contrast plugin against WCAG 2.1 AA ratios

1. Load fonts

The system uses Lato from Google Fonts. Load all weights used in the system:

HTML 
<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Lato:ital,wght@0,300;0,400;0,700;0,900;1,400&display=swap"
      rel="stylesheet">

2. Load CSS tokens

Import colors_and_type.css once in your document. All design tokens are available as CSS custom properties on :root.

HTML 
<link rel="stylesheet" href="path/to/colors_and_type.css">
CSS 
.my-component {
  background-color: var(--color-bg-card);
  color:            var(--color-text-primary);
  font-size:        var(--text-body);
  padding:          var(--card-padding);
  border-radius:    var(--radius-card);
  box-shadow:       var(--shadow-card);
}

3. Load Material Icons

The system uses Material Icons (outlined) as the icon set:

HTML 
<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">

<!-- Usage -->
<span class="material-icons" aria-hidden="true">arrow_forward</span>

4. Page structure

Every page must include a skip link, correct lang attribute, and landmark regions:

HTML 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page title — Service name</title>
</head>
<body>

  <!-- Skip link: first focusable element -->
  <a href="#main-content" class="skip-link">Skip to main content</a>

  <header role="banner">
    <!-- App header -->
  </header>

  <main id="main-content">
    <h1>Page title</h1>
    <!-- Page content -->
  </main>

  <footer>
    <!-- Help panel -->
  </footer>

</body>
</html>

5. Focus ring

Apply the gold GOV.UK-style focus ring globally. This is required for keyboard accessibility:

CSS 
/* Show for keyboard users only */
*:focus-visible {
  outline: 3px solid #FFBF47;
  outline-offset: 2px;
}

/* Remove for mouse/pointer users */
*:focus:not(:focus-visible) {
  outline: none;
}

6. React component library

Pre-built components are in ui_kits/social-housing/. They use Babel standalone and can be embedded directly in HTML or imported into a React project.

FileComponents
Header.jsxAppHeader, HeroHeader
Button.jsxPrimaryButton, SecondaryButton, TertiaryButton, BackLink, Chip
Card.jsxServiceCard, ApplicationCard, InfoSectionCard, StepCard, HelpPanel
FormField.jsxFormField, SelectField, RadioGroup, CheckboxGroup, FormSection
Navigation.jsxBottomNav, BackLink
