Semantic usage

Always reference semantic tokens in component code — not raw hex values. Semantic tokens map intent to the base palette and allow the system to evolve without updating every component.

Accessibility rules for colour

  • check_circle Never use colour as the only means of conveying information. Always pair colour with an icon, label, or text. 1.4.1 A
  • check_circle All normal text must achieve a minimum 4.5:1 contrast ratio against its background. 1.4.3 AA
  • check_circle Large text (18pt+ or 14pt+ bold) requires a minimum 3:1 contrast ratio. 1.4.3 AA
  • check_circle UI component boundaries (input borders, button outlines) require 3:1 contrast against adjacent colours. 1.4.11 AA
  • check_circle Do not use #888888 or lighter greys for text on any light background — it fails at all text sizes. 1.4.3 AA
  • check_circle The warm accent colours (salmon, peach) must not be used for text or interactive elements — they do not meet contrast requirements against white. 1.4.3 AA
  • check_circle Error states must pair the red border with a visible text error message — not colour alone. 1.4.1 A

CSS custom properties

All colour tokens are defined in colors_and_type.css as CSS custom properties on :root.

/* Primary greens */
--color-green-900: #004D44;   /* Deep Forest */
--color-green-800: #008660;   /* Rich Green */
--color-green-750: #00875F;   /* Action Green */
--color-green-700: #028072;   /* Mid Teal */
