Harp mark

The Irish Government "Simplified Digital Harp" appears in the app header — white on green. It is a stylised SVG harp with a vertical rule and government text. It must always appear white on the deep green header; never in a colour variant.

Accessibility requirements

  • check_circle Decorative icons that repeat adjacent text must have aria-hidden="true" to prevent screen reader duplication. 1.1.1 A
  • check_circle Icon-only buttons (notification bell, avatar, close) must have a visible or programmatic label: aria-label="Notifications". 1.1.1 A
  • check_circle Informative icons (status icons, standalone meaning) must have an alt attribute or associated text. 1.1.1 A
  • check_circle All illustrations in the UI are decorative. Use alt="" on <img> elements and aria-hidden="true" on SVGs. 1.1.1 A
  • check_circle Icon-only interactive elements must have a touch target of at least 44×44px regardless of the icon's visual size (24px). 2.5.5
  • check_circle Icons used for status (success, error, warning) must always be paired with a text label — never colour and icon alone. 1.4.1 A

Correct usage patterns

HTML
<!-- Decorative: icon repeats adjacent text -->
<button>
  <span class="material-icons" aria-hidden="true">arrow_forward</span>
  Apply now
</button>

<!-- Icon-only: needs aria-label -->
<button aria-label="Notifications">
  <span class="material-icons" aria-hidden="true">notifications</span>
</button>

<!-- Informative illustration -->
<img src="illustration-house.png" alt="" aria-hidden="true">

<!-- Status icon: pair with text -->
<span class="material-icons" aria-hidden="true">check_circle</span>
<span>Application submitted</span>

Using icons

HTML
<!-- Standard 24px icon -->
<span class="material-icons" aria-hidden="true">arrow_forward</span>

<!-- Size override -->
<span class="material-icons"
      style="font-size: 32px"
      aria-hidden="true">home</span>
